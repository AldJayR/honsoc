# Sprint 2.5 — Email Infrastructure: Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the console.log email mock with real transactional email delivery using Resend + React Email templates, with pg-boss as the background job queue.

**Architecture:** `sendEmail()` enqueues a job to pg-boss (backed by the existing PostgreSQL DB). A pg-boss worker picks up the job, looks up the user's name, renders the appropriate React Email template, and sends via Resend. This decouples email sending from the request cycle and provides automatic retries.

**Tech Stack:** pg-boss (job queue), Resend (email delivery), React Email (template rendering), Fastify 5, Drizzle ORM, TypeScript

### Task 5: Rewrite `sendEmail`

**Files:**
- Modify: `src/lib/email.ts`

Replace the console.log mock with pg-boss job enqueue + user name lookup.

```typescript
// src/lib/email.ts
import { eq } from "drizzle-orm";
import { db } from "@/db/index.ts";
import { users } from "@/db/schema/index.ts";
import { boss } from "@/lib/queue.ts";

export type EmailTemplate = "verify-email" | "reset-password" | "invite-officer";

interface SendEmailJob {
	to: string;
	template: EmailTemplate;
	props: {
		userName: string;
		url: string;
	};
}

export async function sendEmail(
	to: string,
	template: EmailTemplate,
	url: string,
): Promise<void> {
	// Look up user name for personalization
	const user = await db.query.users.findFirst({
		where: eq(users.email, to),
		columns: { name: true },
	});

	const userName = user?.name ?? "User";

	await boss.send("emails", {
		to,
		template,
		props: { userName, url },
	} satisfies SendEmailJob);
}
```

**Key changes:**
- Signature: `(to, subject, html)` → `(to, template, url)` — callers no longer pass raw HTML
- Looks up user name from DB (same connection, no extra pool)
- Enqueues to `emails` queue with typed payload

**Update callers** — `src/auth/index.ts`:

### Task 6: Queue Worker

**Files:**
- Create: `src/lib/worker.ts`

The worker registers a pg-boss `work()` handler for the `emails` queue. When a job arrives, it renders the React Email template and sends via Resend.

```typescript
// src/lib/worker.ts
import { boss } from "@/lib/queue.ts";
import { resend } from "@/lib/resend.ts";
import { env } from "@/config/env.ts";
import { VerifyEmail } from "@/lib/emails/verify-email.tsx";
import { ResetPassword } from "@/lib/emails/reset-password.tsx";
import { InviteOfficer } from "@/lib/emails/invite-officer.tsx";
import type { EmailTemplate } from "@/lib/email.ts";

const templates: Record<EmailTemplate, (props: { userName: string; url: string }) => React.ReactNode> = {
	"verify-email": VerifyEmail,
	"reset-password": ResetPassword,
	"invite-officer": InviteOfficer,
};

export async function registerWorker() {
	await boss.work("emails", async ([job]) => {
		const { to, template, props } = job.data as {
			to: string;
			template: EmailTemplate;
			props: { userName: string; url: string };
		};

		const reactTemplate = templates[template];
		if (!reactTemplate) {
			throw new Error(`Unknown email template: ${template}`);
		}

		const { error } = await resend.emails.send({
			from: env.RESEND_FROM,
			to: [to],
			subject: getSubject(template),
			react: reactTemplate(props) as React.ReactNode,
		});

		if (error) {
			throw new Error(`Resend error: ${error.message}`);
		}
	});
}

function getSubject(template: EmailTemplate): string {
	switch (template) {
		case "verify-email":
			return "Verify your email address";
		case "reset-password":
			return "Reset your password";
		case "invite-officer":
			return "You've been invited to NHSVS";
	}
}
```

**Key points:**
- Worker is registered after pg-boss starts
- Throws on Resend error → pg-boss retries the job (default: 3 retries with exponential backoff)
- `react:` parameter receives the component as a function call result (per Resend docs)
- Subject is derived from template name — keeps the job payload small

### Task 7: Lifecycle Wiring

**Files:**
- Modify: `src/index.ts`

Start pg-boss + register worker before Fastify listens. Stop pg-boss on shutdown.

```typescript
// src/index.ts
import "dotenv/config";
import { env } from "@/config/env.ts";
import { buildApp } from "@/app.ts";
import { setupStorageCors } from "@/lib/storage.ts";
import { startQueue, stopQueue } from "@/lib/queue.ts";
import { registerWorker } from "@/lib/worker.ts";

async function main() {
	await setupStorageCors();
	await startQueue();
	await registerWorker();

	const app = await buildApp();
	await app.listen({ port: env.PORT, host: env.HOST });

	let closing = false;
	const signals = ["SIGINT", "SIGTERM"] as const;
	for (const signal of signals) {
		process.on(signal, async () => {
			if (closing) return;
			closing = true;
			await app.close();
			await stopQueue();
			process.exit(0);
		});
	}
}

main().catch((err) => {
	console.error(err);
	process.exit(1);
});
```

**Startup order:** dotenv → setupStorageCors → startQueue → registerWorker → buildApp → listen
**Shutdown order:** app.close (drain HTTP) → stopQueue (drain jobs) → exit
